Project Goal

This project implements an event-driven, metadata-based Azure Data Factory pipeline
that automatically ingests, validates, routes, and transforms files stored in Azure Blob Storage.

The pipeline is triggered by storage events and dynamically decides how files should be processed
based on metadata and naming rules. Only selected datasets (e.g. files starting with "tita")
are routed to transformation logic, ensuring data quality and efficient processing.

The main goal of the project was to demonstrate cloud-native Data Engineering practices:

  • event-driven ETL orchestration
  • metadata-driven pipeline design
  • dynamic branching using ForEach & If Condition
  • reusable, parameterized datasets
  • scalable storage-only data pipelines

Architecture Overview

High-level flow:

  1. File is uploaded to Azure Blob Storage
  2. Storage event automatically triggers the ADF pipeline
  3. File metadata is read and evaluated
  4. Dynamic routing logic decides processing path
  5. Eligible files are transformed and written to curated storage
  6. Trigger files are deleted after successful processing

This architecture ensures fully automated, reliable, and scalable ingestion.

Master Pipeline Event-Driven Orchestration

The master pipeline is executed automatically via a storage event trigger
and coordinates the entire workflow.

Responsibilities:

  • React to file upload events
  • Copy incoming files into staging layers
  • Execute downstream pipelines
  • Remove trigger files only after successful execution

This guarantees idempotent and hands-free processing.

Executed Pipeline Metadata-Driven Logic

The executed pipeline contains the core dynamic logic.

Processing steps:

  • Get Metadata reads file name and attributes
  • ForEach Activity iterates over detected files
  • If Condition applies naming-based logic

Conditional Routing If Activity

Files that start with "tita" are processed further,
while non-matching files are skipped or handled separately.

This prevents unnecessary processing and enforces data quality rules.

Parameterized Datasets

All datasets are fully parameterized, enabling:

  • dynamic source paths
  • dynamic sink locations
  • reuse across pipelines
  • easy extension to new datasets

This design significantly improves maintainability and scalability.

Data Transformation Mapping Data Flow

Selected files are transformed using ADF Mapping Data Flows, where data is:

  • transformed according to business rules
  • written back to curated storage layers

Transformation logic is fully decoupled from orchestration.
